    40‚Üí  1. `src/app/api/social/profile/route.ts` ‚Äî POST upserts PublicProfile (uid, displayName, photoURL) and returns the profile JSON.
    41‚Üí  2. `src/app/api/social/following/route.ts` ‚Äî GET `?uid=` returns array of followingUid strings (Follow.findMany select followingUid).
    42‚Üí  3. `src/app/api/social/follow/route.ts` ‚Äî POST upserts placeholder PublicProfiles for both currentUid and targetUid (FK safety) then upserts Follow (followerUid=currentUid, followingUid=targetUid). DELETE uses deleteMany on the same pair. Both return `{ ok: true }`.
    43‚Üí  4. `src/app/api/social/search/route.ts` ‚Äî GET `?q=` searches PublicProfile.displayName via `contains` (no insensitive mode for SQLite), limits 20, returns `{ uid, displayName, photoURL }`.
    44‚Üí  5. `src/app/api/social/feed/route.ts` ‚Äî GET returns 50 most recent FeedPosts ordered createdAt desc with all fields, createdAt as ISO. POST upserts the author PublicProfile first (FK), creates the FeedPost, returns created post with createdAt ISO.
    45‚Üí  6. `src/app/api/social/kudos/route.ts` ‚Äî POST `{ postId }` increments kudosCount atomically, returns `{ ok: true, kudosCount }`.
    46‚Üí  7. `src/app/api/social/comments/route.ts` ‚Äî GET `?postId=` returns comments ordered createdAt asc with ISO dates. POST reads author identity from `x-user-uid` / `x-user-name` / `x-user-photo` headers (defaults to "Local Athlete" / "local-athlete" uid), creates Comment, increments FeedPost.commentCount. DELETE removes comment and decrements commentCount, clamped at 0 (re-read + patch).
    47‚Üí  8. `src/app/api/challenges/route.ts` ‚Äî GET counts active challenges (startDate <= now && endDate >= now); if 0, upserts the 3 default challenges (centurion_volume 10k, iron_titan 50k, hypertrophy_hero 100k) with startDate=7 days ago, endDate=30 days from now. Then returns all active challenges ordered by goalKg asc as `{ id, title, description, goalKg, startDate, endDate }`.
    48‚Üí  9. `src/app/api/challenges/[challengeId]/join/route.ts` ‚Äî POST with `{ params }: Promise<{ challengeId: string }>` signature (Next.js 16 awaited-params). Upserts Participation on `(challengeId, userId)`; new rows get progressKg=0, completed=false; refreshes userName/userPhotoURL on re-join. Returns participation with completedAt (ISO or null) and joinedAt (ISO).
    49‚Üí  10. `src/app/api/challenges/[challengeId]/progress/route.ts` ‚Äî GET `?userId=` returns the Participation for `(challengeId, userId)` or null if not joined. Same shape as join response.
    50‚Üí  11. `src/app/api/challenges/[challengeId]/leaderboard/route.ts` ‚Äî GET returns top 100 Participations ordered progressKg desc with all dates as ISO strings.
    51‚Üí  12. `src/app/api/challenges/sync-volume/route.ts` ‚Äî POST `{ userId, totalVolume }` finds all in-progress Participations for the user (with their challenge.goalKg), adds totalVolume to progressKg, marks completed=true with completedAt=now when goalKg is met. Returns `{ ok: true, updated: count }`. Treats non-positive volume as a no-op.
